# SRQ2: Multi-Agent Architecture & Coordination

## Research Question

**How can a multi-agent architecture coordinate predictive models and heterogeneous data signals to generate actionable managerial recommendations?**

## Scope

This notebook focuses on understanding:
1. **Multi-agent systems**: Architecture patterns, coordination mechanisms
2. **Heterogeneous data signals**: Combining time series, categorical, textual, structural data
3. **Agent coordination**: How agents communicate, aggregate, resolve conflicts
4. **Decision generation**: From model outputs to actionable recommendations
5. **Explainability**: How to explain agent decisions to business stakeholders

## Paper Selection Criteria

Include papers on:
- Multi-agent reinforcement learning and coordination
- Ensemble methods and voting mechanisms
- Hierarchical and distributed architectures
- Data fusion and heterogeneous data integration
- Large language models (LLMs) as agent orchestrators
- Multi-agent systems for decision-making
- Knowledge graphs and semantic integration
- Federated learning architectures
- Agent communication protocols
- Workflow orchestration and DAGs

Exclude papers on:
- Single-agent systems (not directly about coordination)
- Purely theoretical game theory without practical application
- Multi-agent systems for non-decision-support tasks (e.g., robotics)

## Key Concepts to Track

When reading papers, look for:
- **Coordination mechanisms**: How do agents share information?
- **Scalability**: How many agents? How much data?
- **Latency**: What's the time-to-decision?
- **Conflict resolution**: How are disagreements handled?
- **Explainability**: Can you explain why recommendations were made?
- **Learning**: Do agents improve over time?
