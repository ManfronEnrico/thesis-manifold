## 2.0 Chapter Introduction

This chapter reviews the academic literature across five thematic research angles that collectively constitute the theoretical foundation of this thesis. The review is structured to follow the logical progression from the enabling infrastructure of AI agent systems, through predictive modelling approaches and their resource constraints, to the integration and synthesis mechanisms required for decision-support. Each section concludes with an explicit identification of the gap that the thesis addresses, and the chapter closes with a consolidated gap statement and novelty claim grounded in the reviewed corpus.

The literature spans the period from 2010 to 2026, with heavier representation of post-2020 publications in the AI and machine learning sections, reflecting the rapid publication pace in those fields. The review covers 37 confirmed papers across five thematic areas: AI agents and large language model orchestration, predictive modelling for retail decision-support, hybrid AI integrating symbolic reasoning and machine learning, multi-criteria decision synthesis, and resource-constrained AI deployment. Together, these areas define the design space in which this thesis operates and establish the combination of gaps that the proposed framework addresses.

## 2.1 AI Agents and Large Language Model Orchestration

*Maps to Main RQ and SRQ2*

### 2.1.1 From Language Models to Agents

The emergence of large language models (LLMs) as reasoning engines has catalysed a paradigmatic shift in how artificial intelligence systems are designed. Whereas early LLMs were conceived primarily as generative text systems responding to prompts, the agent paradigm reconceptualises them as action-taking entities capable of interacting with external tools, decomposing complex tasks, and revising plans in response to execution feedback. This distinction between an LLM as a language model and an LLM as an agent is foundational to understanding the architectural choices made in this thesis.

A seminal contribution to establishing this shift is Toolformer, in which Schick et al. (2023) demonstrated that language models can learn to invoke external APIs, calculators, and search engines through self-supervised training on API call annotations. Crucially, a 6.7-billion-parameter Toolformer outperformed GPT-3 at 175 billion parameters on a range of downstream reasoning tasks, establishing empirically that the ability to delegate sub-tasks to specialised external tools can substitute for raw parameter scale. For this thesis, this finding provides the foundational justification for designing a system in which the LLM orchestrates specialised machine learning forecasting models rather than attempting to forecast directly: tool delegation, not model scale, is the design principle.

Paranjape et al. (2023) extended this approach through ART (Automatic Multi-Step Reasoning and Tool Use), a framework that automates the decomposition of complex tasks into structured sequences of reasoning steps and tool invocations, without requiring manual workflow engineering. ART demonstrates that structured planning decomposition, derived automatically from task context, enables LLMs to solve multi-step problems more reliably than single-shot prompting. This finding directly informs the thesis synthesis pipeline design, in which the Synthesis Agent executes a five-step processing sequence rather than generating a recommendation in a single unstructured call.

Wang et al. (2024) advanced this line of work by establishing that code execution, rather than JSON-formatted tool calls, is the superior action format for LLM agents. Their framework, which treats Python code as the native action language for agent steps, outperforms JSON-based alternatives by up to 20 percentage points on agentic benchmarks, and enables dynamic tool composition and self-debugging that JSON formats cannot support. This result provides the action-execution rationale for the thesis LangGraph coordinator, which manages agent transitions through typed state updates rather than free-form JSON outputs, and informs the use of Python-native tool interfaces throughout the multi-agent pipeline.

Sapkota et al. (2025) provide the conceptual taxonomy that situates the thesis within the broader landscape of AI systems. Their framework distinguishes between AI Agents, defined as modular, task-specific automation systems driven by a single LLM, and Agentic AI, characterised by multi-agent collaboration, dynamic task decomposition, persistent memory, and coordinated autonomy. Under this taxonomy, the thesis framework qualifies as an Agentic AI system with human-in-the-loop approval gates: it comprises four specialised agents coordinated by a central orchestrator, operates across multiple interdependent processing phases, and incorporates explicit human checkpoints between phases. This classification has implications for how the system is evaluated, as Agentic AI systems require evaluation of coordination logic and emergent behaviour in addition to individual component performance.

### 2.1.2 Multi-Agent Coordination

Single-agent architectures face three well-documented limitations when applied to complex, multi-step tasks: context window overflow, which occurs when the full task context exceeds the LLM's input capacity; role conflict, whereby a single agent must simultaneously act as planner, executor, and critic; and the absence of specialisation, which forces the agent to generalise across domains that would benefit from dedicated expertise. Multi-agent architectures address these limitations through task decomposition, role specialisation, and structured communication between agents.

LangChain AI (2024) provides the orchestration substrate for the thesis through LangGraph, a stateful multi-agent coordination framework built on directed computational graphs with conditional routing. LangGraph implements typed state shared across agents, conditional edge traversal that routes execution based on runtime outputs, and human-in-the-loop interruption at designated approval gates. These features are precisely those required by the thesis framework: shared TypedDict state enables agents to pass structured outputs without uncontrolled side effects, conditional routing enables the Coordinator to redirect execution when validation fails, and interrupt nodes implement the human approval gates that are mandatory at each phase transition.

Liu et al. (2023) demonstrate the value of dynamic agent specialisation through DyLAN (Dynamic LLM-Powered Agent Network), in which specialist agents are selectively activated based on task requirements rather than being executed unconditionally in a fixed sequence. DyLAN achieves 10.3 percentage point improvement over static routing baselines on multi-step reasoning benchmarks, confirming that dynamic, heterogeneous agent networks outperform fixed pipelines when the task requires different competencies at different stages. This finding directly supports the thesis four-agent decomposition, in which each agent, covering data ingestion, forecasting, synthesis, and validation respectively, is activated according to the phase of the pipeline rather than being executed in parallel or unconditionally.

Chen et al. (2024) provide complementary evidence through AutoFlow, a system that automatically generates directed acyclic graph (DAG) workflows for LLM agent pipelines from high-level task specifications. AutoFlow produces workflows that perform within five percentage points of hand-crafted expert pipelines while requiring no manual workflow engineering, and demonstrates that structured graph-based orchestration generalises across task types. Wu et al. (2025) extend this line of work through ScoreFlow, which applies score-based preference optimisation to LLM agent workflows, achieving four to seven percentage point improvements over DSPy and OPRO baselines by using continuous preference signals to refine multi-step coordination. Together, AutoFlow and ScoreFlow establish that the structured, graph-based orchestration approach adopted by the thesis, implemented through LangGraph, reflects best practice in the multi-agent literature.

Ma et al. (2024) demonstrate through SciAgent that equipping LLMs with a curated toolset of domain-specific tools, covering mathematical computation, knowledge retrieval, and specialised APIs, yields 13.4 percentage point improvement over tool-free GPT-4 on structured scientific reasoning tasks. SciAgent reinforces the core design principle of this thesis: that LLM agents acting as reasoning orchestrators, delegating numerical computation to dedicated tools while retaining synthesis and communication responsibilities, outperform end-to-end LLM approaches on precision-sensitive tasks. In the thesis context, this principle translates to delegating demand forecasting to specialised ML models while retaining the LLM for recommendation synthesis, confidence explanation, and natural language generation.

### 2.1.3 Reliability and Limitations of LLM Agents

The deployment of LLM agents in production settings introduces reliability challenges that the research literature has begun to systematically characterise. These challenges are especially consequential in business decision-support contexts, where LLM-generated outputs influence managerial decisions with real operational and financial consequences. Three categories of reliability risk are particularly relevant to this thesis: hallucination, input noise sensitivity, and coordination failure.

Ji et al. (2024) establish a systematic taxonomy of LLM hallucination through ANAH, an analytical annotation framework that classifies factual errors in LLM outputs across a structured hierarchy of error types. Their analysis documents that hallucination in LLM outputs is not a random phenomenon but follows identifiable patterns, including entity substitution, numerical imprecision, and unsupported causal attribution. For this thesis, the ANAH taxonomy motivates the mandatory validation step in which the Synthesis Agent's outputs are verified against the source forecast values before they reach the user: the thesis design treats LLM hallucination on numerical data as a documented, quantified risk rather than a theoretical possibility.

Wang et al. (2026) extend this analysis to tool-using LLM agents under noisy input conditions through AgentNoiseBench, demonstrating that agent performance degrades systematically when tool inputs contain structured noise, such as mislabelled features or formatting inconsistencies. The benchmark documents degradation patterns across multiple agent families and noise types, establishing that noise robustness is a measurable, non-trivial engineering challenge in production agent deployments. This finding informs the thesis data preprocessing design, in which Nielsen scanner data and Indeks Danmark consumer survey features are validated and standardised before they are passed as inputs to the Synthesis Agent, specifically to prevent the noise-induced degradation documented by Wang et al. (2026).

Sapra et al. (2025) provide empirical evidence of coordination failure in production agentic workflows through AgentCompass, a benchmarking study that evaluates agentic workflow performance against production expectations. Their findings reveal a 34 percent step-level error rate relative to benchmark expectations in unstructured agentic deployments, and demonstrate that structured traceability mechanisms reduce debugging time by 60 percent when failures occur. Huang et al. (2024) contribute a complementary mechanism: self-verification sampling, in which the LLM generates and evaluates candidate outputs before committing to a tool call, reduces argument errors by 31 percent and improves task completion by 8.4 percentage points without requiring fine-tuning. Together, these findings motivate the thesis validation architecture, in which mandatory human approval gates serve as coordination failure checkpoints, and the Validation Agent independently verifies Synthesis Agent outputs before they are delivered to the user.

### 2.1.4 Gap in This Section

The reviewed literature on AI agents and LLM orchestration establishes a robust theoretical and empirical foundation for multi-agent system design, but reveals a consistent limitation: virtually all agent architectures in this literature assume cloud-scale compute infrastructure, with no systematic treatment of memory constraints as a binding design variable. DyLAN, AutoFlow, ScoreFlow, and SciAgent each evaluate their systems on GPU-enabled infrastructure capable of loading multi-billion-parameter models into memory; none report RAM budgets or model memory footprints as design-relevant metrics. Similarly, González-Potes et al. (2026), whose hybrid LLM plus ML architecture for industrial process supervision is the architecturally closest published system to the thesis framework, explicitly note that their deployment is designed for continuous real-time industrial process supervision on dedicated industrial computing infrastructure, and does not address the memory constraints of SME cloud deployments. The thesis addresses this gap by designing and evaluating a multi-agent framework that operates within a total RAM budget of eight gigabytes, treating this constraint as a first-order architectural requirement rather than a post-hoc consideration.

## 2.2 Predictive Modelling for Decision-Support

*Maps to SRQ1 and SRQ4*

### 2.2.1 From Descriptive to Predictive Analytics

Business intelligence systems have historically occupied the descriptive tier of the analytics spectrum, aggregating historical transactional data into dashboards, key performance indicator reports, and trend summaries. This descriptive function has generated substantial operational value for organisations, but the competitive dynamics of modern FMCG markets increasingly demand the ability to anticipate rather than retrospectively report. The transition from descriptive to predictive analytics requires different modelling infrastructure, different data pipelines, and different interaction paradigms between the system and its users.

Rinaldi et al. (2025) document this transition in the context of AI-enabled decision support, proposing DSS4EX, a framework for AI decision support systems applied to time series forecasting that incorporates an explainability layer generating natural language explanations of AI pipeline outputs. Their evaluation demonstrates that the explainability layer yields significantly higher decision quality scores than raw ML outputs presented without explanation, establishing empirically that the interface between predictive models and decision-makers is as important to decision quality as model accuracy. This finding directly motivates the thesis Synthesis Agent design, in which the LLM generates a natural language recommendation accompanied by a numerical confidence score rather than delivering raw forecast values.

Olszak and Bartuś (2025) confirm this direction through a study on AI-enhanced business intelligence, reporting 34 percent higher decision confidence and 28 percent higher forecast adoption rates for AI-enhanced BI systems relative to traditional descriptive BI, with the primary driver being the co-presentation of explanations alongside predictions. Herath et al. (2024), conducting action design research on AI-augmented decision-making at three organisations, derive a design principle with direct relevance to the thesis: AI systems must communicate uncertainty to be trusted by non-technical business users. Both findings are directly operationalised in the thesis framework through the confidence score and natural language recommendation generated by the Synthesis Agent.

### 2.2.2 Forecasting Approaches for Retail Demand

The retail FMCG forecasting literature has converged on a set of modelling approaches whose relative performance depends critically on the characteristics of the demand series being forecast: its temporal regularity, the availability of exogenous features, the size of the training set, and the computational resources available at inference time. The five models selected for the thesis benchmark, specifically ARIMA, Prophet, LightGBM, XGBoost, and Ridge Regression, span the spectrum from classical statistical approaches to gradient-boosted tree ensembles, and were selected to provide comprehensive coverage of the accuracy-efficiency trade-off frontier under the eight-gigabyte RAM constraint.

Ceran et al. (2024), in a study on machine learning-based demand forecasting for an FMCG retailer conducted in collaboration with Migros, Turkey's largest grocery chain, establish LightGBM as the method achieving the best accuracy at the lowest RAM footprint in a real retail setting, and propose a MAPE target of 15 percent or below as an industry benchmark for acceptable forecasting performance. Ma et al. (2025) extend this finding to the beverage manufacturing context, demonstrating through a large-scale empirical study of a private-label beverage manufacturer that machine learning models enriched with exogenous contextual features substantially outperform statistical baselines for high-volume stable SKUs, with no single model dominating across all demand patterns. These two findings jointly motivate the thesis benchmark design: LightGBM is included as the leading candidate for the resource-constrained retail setting, and the multi-model evaluation framework reflects the absence of a universally dominant model.

The M4 Competition results, analysed by Makridakis et al. (2020) across 100,000 time series and 61 forecasting methods, established that combining multiple forecasting models consistently outperforms any single best model, and that hybrid methods blending statistical structure with machine learning achieve the highest accuracy. The M5 Competition (Makridakis et al., 2022), focused specifically on hierarchical retail sales forecasting using Walmart scanner data, reinforced these findings in a domain directly relevant to the thesis: all 50 top-performing submissions used LightGBM, achieving over 14 percent improvement relative to the best statistical benchmark, and submissions incorporating exogenous promotional and calendar features consistently outperformed those using sales history alone. Together, the M4 and M5 results provide the empirical basis for three design choices made in this thesis: multiple models are benchmarked rather than a single model selected a priori, LightGBM is included as a primary candidate, and external features are incorporated as enrichment signals.

Ahrens et al. (2024) provide the theoretical grounding for multi-model combination through their work on model averaging and double machine learning (DML). Their analysis demonstrates that stacked model averaging, particularly inverse-variance weighting, systematically improves prediction accuracy over any individual learner in the ensemble, and that the DML framework provides an orthogonalisation approach that separates nuisance parameter estimation from target parameter estimation. For the thesis, this theoretical result supports the ensemble weighting step implemented in the Synthesis Agent, in which outputs from multiple forecasting models are combined using a weighting scheme that accounts for individual model confidence rather than assigning equal weights.

Nguyen et al. (2025) confirm the practical applicability of gradient-boosted ensemble methods in resource-limited contexts through a comprehensive review of over 120 machine learning papers in economic forecasting and SME growth applications. Their review identifies LightGBM and XGBoost as optimal for short-horizon forecasting with fewer than 500 training observations, which precisely characterises the thesis setting with approximately 36 monthly Nielsen periods. This finding reinforces the thesis model selection rationale: LightGBM and XGBoost are not included in the benchmark merely because they are popular methods, but because the empirical literature specifically validates their performance in the short-horizon, limited-data regime that the thesis confronts.

### 2.2.3 Resource-Constrained Model Deployment

The deployment of machine learning forecasting models in production environments requires attention not only to predictive accuracy but to computational efficiency, measured across three dimensions: peak RAM consumption at inference time, runtime per forecasting cycle, and output stability across repeated runs with identical inputs. These efficiency dimensions have received comparatively limited attention in the academic forecasting literature, which has traditionally evaluated models exclusively on accuracy metrics. Two recent contributions begin to address this gap.

Liu et al. (2025) evaluate resource optimisation strategies for edge AI deployments, demonstrating that quantisation and knowledge distillation achieve 94 to 97 percent of full-precision accuracy at three to five times speedup on ARM-class hardware. Their results establish the measurability of the accuracy-efficiency trade-off and confirm that systematic RAM profiling is feasible as a design-validation exercise rather than merely as an operational monitoring task. Semerikov et al. (2025) extend this analysis to the deployment of large language models in resource-constrained environments, documenting that INT8 quantisation combined with knowledge distillation dominates practical deployment strategies. The key finding for this thesis is the implicit confirmation that LLMs used for synthesis and orchestration, accessed via API rather than loaded locally, represent the only viable LLM deployment pattern under an eight-gigabyte RAM budget: local LLM inference is eliminated as an option by the constraint, making API-based synthesis the architecturally necessary choice.

Klee and Xia (2025) address a dimension of production deployment that the broader forecasting literature has largely neglected: forecast stability across repeated runs. Their study defines stability as the coefficient of variation of forecasts across consecutive production cycles with nominally identical inputs, and documents substantial variation across model families, with tree ensemble methods exhibiting greater stability than classical statistical models under promotional data conditions. Klee and Xia (2025) explicitly call for further research on cycle-to-cycle stability as a production deployment metric, noting that a model with marginally higher accuracy but unstable output is a worse operational choice than a slightly less accurate but consistent alternative. This thesis directly responds to that call by incorporating stability as a third evaluation dimension alongside accuracy and memory efficiency in the SRQ1 benchmark.

### 2.2.4 Gap in This Section

The predictive modelling literature reviewed in this section reveals two specific gaps that the thesis addresses. First, no published study systematically benchmarks ARIMA, Prophet, LightGBM, XGBoost, and Ridge Regression head-to-head within a strict RAM budget in a retail CPG context, measuring all three dimensions of accuracy, computational efficiency, and stability simultaneously. Individual models are evaluated in isolation across multiple studies, but the tri-dimensional comparative benchmark does not exist for the retail FMCG setting. Second, the M5 findings regarding the value of exogenous features (Makridakis et al., 2022) are confined to promotional and calendar variables. The thesis extends this finding to a qualitatively different class of external signal, namely consumer attitudinal data collected through a dedicated national survey, which operates at a different temporal resolution and semantic level than transactional records.

## 2.3 Hybrid AI: Integrating Symbolic Reasoning and Machine Learning

*Maps to SRQ2 and Main RQ*

### 2.3.1 Neuro-Symbolic AI

The integration of symbolic reasoning with data-driven machine learning represents one of the most actively debated architectural paradigms in contemporary AI research. Symbolic AI approaches, including rule-based systems, logic programming, and constraint solvers, offer interpretability, compositionality, and the ability to enforce domain constraints, but struggle to learn from raw data. Neural approaches, conversely, excel at pattern recognition in high-dimensional data but are opaque and do not reliably enforce logical constraints. Hybrid neuro-symbolic architectures seek to combine the complementary strengths of both paradigms.

Colelough and Regli (2025) provide the most comprehensive recent synthesis of this literature through a systematic PRISMA review covering over 200 papers on neuro-symbolic AI. Their analysis establishes that neuro-symbolic systems outperform pure neural architectures by 15 to 30 percentage points in low-data regimes containing fewer than 1,000 training examples, and propose a taxonomy of integration patterns ranging from loose coupling, where symbolic and neural components share only outputs, to tight coupling, where symbolic constraints are embedded within the neural training process. Under this taxonomy, the thesis framework qualifies as a loosely coupled hybrid: the machine learning forecasting models operate independently as purely data-driven components, while the LLM Synthesis Agent applies structured reasoning over their outputs according to an explicitly defined confidence scoring formula. This characterisation is theoretically significant because it implies that the thesis system's behaviour is partially predictable from the symbolic reasoning rules, even when the ML models' outputs vary.

Mehdi et al. (2024) propose a more formal integration framework for symbolic reasoning and deep learning, arguing that hybrid architectures outperform pure approaches in complex decision-making tasks that require both pattern recognition in historical data and logical inference over the resulting predictions. Their framework identifies three categories of integration: pre-hoc, where symbolic knowledge is incorporated into the data preprocessing and feature engineering stage; in-hoc, where symbolic constraints are embedded in the model training objective; and post-hoc, where symbolic reasoning is applied to the outputs of a trained neural model. The thesis implements a post-hoc hybrid: ML models produce probabilistic forecasts, and the LLM applies structured post-hoc reasoning to synthesise those forecasts into a natural language recommendation with an explicit confidence score. This design choice reflects both the eight-gigabyte RAM constraint, which precludes end-to-end hybrid training, and the requirement for interpretable, auditable recommendation generation.

### 2.3.2 LLM Plus Machine Learning Integration Patterns

The architectural pattern of combining ML numerical prediction with LLM synthesis has emerged as a distinct design paradigm in applied AI systems, supported by converging empirical evidence across multiple domains. The core pattern is as follows: specialised ML models produce calibrated numerical predictions; these predictions are passed as structured inputs to an LLM, which synthesises them into a contextualised natural language output that is interpretable by non-technical users. This pattern addresses the complementary limitations of each component: ML models produce accurate but opaque numerical outputs that require expert interpretation, while LLMs produce fluent natural language but cannot reliably perform the statistical inference that generates the numerical predictions.

González-Potes et al. (2026) provide the architecturally closest published realisation of this pattern in their hybrid LLM plus ML agent system for clean-in-place industrial process supervision. Their three-layer architecture combines physics-based symbolic rules, ML prediction of process outcomes, and LLM-generated natural language recommendations for process operators. In production evaluation, the system achieves specification compliance rates above 98 percent and median LLM numerical errors below three percent, validating that the LLM plus ML synthesis pattern can meet production reliability standards when deterministic and generative components are carefully separated. The thesis draws directly on this architectural blueprint while transposing it to the retail CPG domain, substituting physics-based symbolic rules with structured confidence scoring formulas and replacing process supervision with demand forecasting.

Zheng et al. (2025) confirm the generalisability of this architectural pattern beyond the industrial process domain through a case study in which an LLM acts as an interpreter of ML outputs in a supply chain planning context. Their study reports that 78 percent of LLM-generated recommendations based on structured ML inputs are rated as actionable by domain experts, compared to 45 percent for raw ML outputs presented without synthesis. This 33 percentage point improvement in recommendation actionability quantifies the value of the LLM synthesis layer, providing empirical support for the thesis design choice of including a Synthesis Agent rather than delivering raw forecasts directly to business users.

### 2.3.3 Gap in This Section

The reviewed literature on hybrid AI establishes the LLM plus ML integration pattern as empirically validated, but the existing realisations share a common limitation: none of them address the combination of retail CPG demand forecasting, consumer survey signal enrichment, and resource-constrained deployment within a single integrated framework. González-Potes et al. (2026) address industrial process control under real-time constraints but not RAM budgets or external consumer data. Zheng et al. (2025) address supply chain orchestration but not the RAM-constrained forecasting benchmark problem. Furthermore, confidence scoring in hybrid AI recommendations is notably absent from all reviewed papers: none of the hybrid AI architectures reviewed propose a composite confidence score formula that combines prediction interval width, cross-model agreement, and external signal alignment into a single numerical confidence indicator. This composite confidence score is the thesis's primary methodological contribution to the LLM plus ML integration pattern.

## 2.4 Multi-Criteria Decision Synthesis

*Maps to SRQ2 and SRQ3*

### 2.4.1 Multi-Criteria Decision-Making Foundations

Multi-Criteria Decision-Making (MCDM) refers to the class of methods for aggregating multiple heterogeneous evaluation criteria, potentially conflicting and expressed in different units, into a single actionable decision or ranking. MCDM methods have been applied extensively in operations management, engineering design, and supply chain planning, and have recently begun to appear in the AI literature as frameworks for combining multiple model outputs into synthesis scores. For this thesis, MCDM provides the theoretical basis for the composite confidence score that aggregates prediction interval width, cross-model agreement, and consumer signal alignment into a single numerical confidence indicator.

Pelissari et al. (2025) provide an overview of MCDM methods in industrial environments, documenting that the Analytic Hierarchy Process (AHP) and TOPSIS account for over 60 percent of industrial applications, and that hybrid MCDM plus ML combinations represent an emerging but rare pattern. Their review establishes that weighted aggregation of heterogeneous criteria is the dominant practical approach, providing methodological grounding for the thesis confidence score formula, which applies pre-defined weights to three independent evaluation dimensions. Aboutorab et al. (2023) demonstrate the direct applicability of this approach in a supply chain context, integrating ML risk prediction scores with AHP-weighted MCDM synthesis to produce explainable supplier selection recommendations. The parallel with the thesis is explicit: ML outputs, representing supplier risk in Aboutorab et al. (2023) and forecast values in this thesis, are passed through a weighted synthesis step to produce a ranked, explainable recommendation. The thesis adapts this pattern to the retail FMCG domain by replacing binary supplier selection with continuous confidence-scored demand recommendations.

Al-Masri et al. (2025) provide an application of stacked ensemble synthesis in the clinical decision support domain, combining five base classifiers with a logistic regression meta-learner to achieve 97.2 percent classification accuracy. While the clinical domain differs substantially from FMCG retail forecasting, the two-tier stacking principle, in which base model outputs are aggregated by a higher-level synthesis layer, is directly transferable. The thesis implements this principle through the Synthesis Agent, which treats the outputs of five base forecasting models as inputs to an LLM-orchestrated synthesis step rather than a second-tier statistical meta-learner, preserving the two-tier structure while replacing the statistical aggregator with a semantically richer synthesis mechanism.

### 2.4.2 Confidence Calibration as a Decision-Quality Mechanism

The practical value of prediction intervals as a communication mechanism for forecast uncertainty is well established in the decision-support literature. Goodwin et al. (2010) demonstrate through a controlled experimental study involving 84 practitioners that forecasts expressed as prediction intervals reduce newsvendor cost by 14.2 percent relative to point forecasts, with the benefit concentrated among high-uncertainty items. This finding provides the primary empirical justification for the thesis generating prediction intervals alongside point forecasts, rather than communicating only the central estimate of future demand.

The practical value of prediction intervals depends critically on their calibration: an interval that nominally covers 80 percent of outcomes but empirically covers only 60 percent misleads decision-makers by providing false precision. Kuleshov et al. (2018) address this calibration problem through isotonic regression-based post-hoc calibration of prediction intervals for deep learning models, establishing that empirical interval coverage can be brought into alignment with stated coverage probabilities through a simple non-parametric post-processing step applied after model training. Levi et al. (2022) confirm that isotonic regression is the most consistently effective calibration method across 12 datasets and six calibration alternatives, including for tree ensemble models such as LightGBM and XGBoost. This confirmation is particularly relevant to the thesis because it validates the applicability of the Kuleshov et al. (2018) calibration protocol to the specific model family that the SRQ1 benchmark identifies as optimal for the Danish CSD context.

### 2.4.3 Contextual Information in Decision Synthesis

The value of external contextual signals as enrichment inputs to demand forecasting systems has been established in the literature but remains underexplored in multi-agent AI contexts. The M5 Competition results (Makridakis et al., 2022) demonstrate that promotional and calendar variables materially improve retail sales forecasting accuracy. Ma et al. (2025) extend this to the beverage sector, showing that consumer contextual features improve forecast performance for stable, high-volume SKUs. SRQ3 of this thesis asks whether consumer attitudinal and behavioural data, collected through the Indeks Danmark national consumer survey, provides a further increment of predictive value beyond transactional sales history alone, extending the M5 finding to a qualitatively different class of external signal.

Kasem et al. (2023) demonstrate that consumer segment features, derived from attitudinal and demographic profiling, are effective enrichment signals for retail demand prediction through customer profiling and sales prediction in a direct marketing context, providing theoretical support for the thesis approach of mapping Indeks Danmark consumer segments to retailer-level demand indicators. Stylianou and Pantelidou (2025) further demonstrate that external consumer attributes improve forecast accuracy for retail demand in a supermarket analytics context, although neither study integrates such signals into a multi-agent AI system or evaluates their marginal contribution in a controlled experimental design. The thesis addresses this gap by comparing forecasting performance with and without Indeks Danmark enrichment features under otherwise identical conditions, providing the controlled evaluation of contextual signal value that the existing literature lacks.

### 2.4.4 Gap in This Section

No paper in the reviewed corpus applies calibrated MCDM-style synthesis to aggregate the outputs of competing ML forecasting models in a retail CPG context. The MCDM literature applies weighting to operational criteria in engineering and supply chain settings; the ML forecasting literature produces numerical outputs without synthesis mechanisms. The gap is precisely at the intersection of these two bodies of work, and it is at this intersection that the thesis makes its primary synthesis contribution. Furthermore, the integration of consumer survey data, specifically behavioural and attitudinal data collected through a dedicated national survey, into ML-based demand forecasting agent systems has not been reported in the reviewed literature, confirming that this is a genuine empirical gap that the thesis is positioned to address.

## 2.5 Resource-Constrained AI Deployment

*Maps to SRQ1 and Main RQ*

### 2.5.1 The Resource Constraint Problem

The deployment of AI systems in real-world business settings is subject to resource constraints that the academic literature has largely not incorporated as first-order design variables. Enterprise cloud deployments capable of running large deep learning models at scale are economically inaccessible to small and medium-sized AI providers, for whom a ceiling of approximately eight gigabytes of total RAM represents not a worst-case assumption but a realistic budget for a commercially deployed product. Ng (2017), analysing four terabytes of Nielsen weekly scanner data, demonstrated empirically that memory constraints are the primary binding design variable in retail scanner data analysis: even with unlimited financial resources, the full dataset cannot be loaded simultaneously, making memory-efficient algorithmic choices necessary rather than merely convenient. For the thesis, Ng (2017) provides the empirical precedent for treating the eight-gigabyte constraint as a first-order architectural requirement derived from the data domain, not as an artificial limitation.

The edge AI literature frames an analogous constraint problem at the hardware level. Liu et al. (2025) evaluate quantisation and distillation strategies for edge AI deployments, establishing that substantial accuracy can be preserved at dramatically reduced memory footprints. Semerikov et al. (2025) synthesise the broader landscape of resource-constrained LLM deployment, documenting that INT8 quantisation and knowledge distillation represent the dominant practical strategies for reducing LLM inference costs, but acknowledging that even aggressively compressed LLMs require memory footprints in the range of one to four gigabytes. For a multi-component pipeline that must simultaneously load data, execute ML models, and maintain agent state, the residual memory budget available for LLM inference at the eight-gigabyte ceiling is insufficient for local model loading. This makes API-based LLM access, the approach adopted in this thesis, the only architecturally viable option for LLM-based synthesis within the defined RAM constraint.

### 2.5.2 Memory Optimisation Strategies

Within the eight-gigabyte RAM budget, the thesis framework adopts three primary memory optimisation strategies derived from the reviewed literature and from first-principles analysis of the pipeline's component requirements. First, sequential rather than parallel model execution is adopted for the forecasting phase: models are loaded, executed, and unloaded one at a time rather than simultaneously, reducing peak memory from the sum of all model footprints to the maximum of any individual model footprint. This approach increases total runtime relative to parallel execution but is the only strategy that keeps peak RAM within the budget when the five candidate models are evaluated. Second, lazy loading is applied throughout the data processing phase: the Nielsen scanner data and Indeks Danmark features are loaded into memory only as required for each processing step and released immediately thereafter. Third, the Indeks Danmark consumer survey, which covers 6,364 variables across 20,134 respondents, is reduced through feature selection to approximately 50 to 100 demand-relevant variables, reducing the memory footprint of the consumer signal enrichment step from an estimated 900 megabytes to approximately 15 megabytes. The RAM profiling methodology used to validate these strategies across the five candidate models is documented in Chapter 6 and constitutes a replicable protocol for multi-component AI pipeline memory auditing.

### 2.5.3 Gap in This Section

The resource-constrained AI deployment literature addresses memory optimisation at the level of individual models and components, but no paper in the reviewed corpus provides a systematic memory profiling methodology for a multi-component AI pipeline that combines ML forecasting and LLM synthesis within a defined total RAM budget. Liu et al. (2025) profile individual edge AI models; Semerikov et al. (2025) profile LLM deployment strategies; but the problem of jointly profiling an ensemble of ML models plus an orchestration layer plus a synthesis module operating within a shared memory budget has not been addressed. The thesis contributes this profiling methodology as a standalone, replicable protocol that is applicable beyond the Danish CSD retail context to any multi-component AI pipeline subject to memory constraints.

## 2.6 Academic Gap Statement and Novelty Claim

The five thematic sections reviewed above collectively establish the state of the art across the design space in which this thesis operates. This section consolidates the gaps identified in each section into an integrated gap statement and derives the novelty claim that the thesis addresses.

**Gap 1: RAM-constrained multi-agent forecasting.** No framework in the reviewed literature combines LLM orchestration, ML ensemble forecasting, and MCDM-style synthesis within a total RAM budget of eight gigabytes. All multi-agent frameworks in the LLM orchestration literature address system design without RAM budgets as a constraint. The hybrid LLM plus ML architectures reviewed in Section 2.3 are designed for industrial process supervision on dedicated hardware, not for SME cloud deployments. The resource-constrained AI deployment literature addresses component-level optimisation but not multi-component pipeline design.

**Gap 2: Head-to-head benchmark under RAM budget.** No paper benchmarks ARIMA, Prophet, LightGBM, XGBoost, and Ridge Regression simultaneously under an explicit RAM constraint in retail CPG, measuring accuracy, computational efficiency, and forecast stability as a unified tri-dimensional evaluation framework. Individual model evaluations exist in the FMCG forecasting literature, but the combined benchmark, motivated by the SRQ1 research question, has not been reported for the resource-constrained retail setting.

**Gap 3: Consumer survey enrichment in agent-based forecasting.** Behavioural and attitudinal consumer survey data, as distinct from promotional calendars and aggregate consumer indices, has not been used as a demand forecasting enrichment signal in multi-agent AI systems. The M5 Competition results (Makridakis et al., 2022) establish the value of exogenous variables in retail forecasting, but the external signals studied are operational rather than attitudinal. Indeks Danmark, as a dedicated national consumer survey covering over 6,000 variables for more than 20,000 respondents, represents a qualitatively distinct class of contextual information whose forecasting value has not been empirically evaluated.

**Gap 4: Empirical evaluation of the descriptive-to-predictive BI transition.** The transition from descriptive to predictive analytics in FMCG retail BI systems has been discussed conceptually in the reviewed literature but has not been empirically evaluated against a defined descriptive analytics baseline in a real retail CPG context. SRQ4 of this thesis provides this empirical evaluation by comparing the predictive framework against the current Manifold AI Colleagues system, which operates exclusively at the descriptive level.

**Gap 5: Memory profiling methodology for multi-component AI pipelines.** No paper in the reviewed corpus provides a replicable RAM profiling protocol for a pipeline that combines ML forecasting with LLM synthesis and multi-agent coordination within a shared memory budget. The thesis contributes this methodology as a standalone artefact, applicable beyond the specific thesis context to any AI pipeline subject to memory constraints.

**Novelty claim.** This thesis is the first to design, implement, and evaluate a multi-agent predictive decision-support framework that simultaneously: (a) operates within an eight-gigabyte RAM constraint through sequential model execution and lazy loading; (b) integrates consumer survey data as a contextual enrichment signal for retail FMCG demand forecasting; (c) generates calibrated, confidence-scored natural language recommendations through a structured MCDM-informed synthesis step; and (d) is empirically evaluated against a defined descriptive analytics baseline in a real Danish retail CSD market context. The theoretical contribution takes the form of five generalised design principles derived from the framework evaluation and structured according to the design science research output format proposed by Hevner et al. (2004), constituting a contribution to both the information systems design science literature and the applied machine learning literature.
