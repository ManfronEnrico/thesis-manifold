# System B: Active Thesis Production System

This directory contains **thesis writing tooling and scaffolding** (not the thesis itself).

## Status

✅ **ACTIVE & EXTENSIBLE** — Features can be added, toggled, and modified safely.

## Explanation

System B is the **production infrastructure** that helps you write the thesis:
- **Purpose**: Generate bullet skeletons, check compliance, validate prose, manage figures
- **Output**: Markdown bullets, JSON state, compliance reports (invisible scaffolding)
- **Extensibility**: Designed for safe feature additions via toggle-gated enhancements
- **In the thesis**: System B never appears; it's behind-the-scenes tooling

## What System B Does

| Component | Purpose |
|-----------|---------|
| `agents/writing_agent.py` | Generates bullet skeletons (never prose) |
| `agents/compliance_agent.py` | CBS formal checks (APA 7, structure, etc.) |
| `agents/literature_agent.py` | Paper corpus management + scraping |
| `agents/diagram_agent.py` | Thesis figure generation |
| `core/coordinator.py` | Orchestration: Plan → Execute → Critic → Persist |
| `state/thesis_state.py` | Pydantic ThesisState (type-safe, extensible) |

## How to Extend System B

System B supports **toggle-gated feature additions**:

1. Add new field to `ThesisState` (backward-compatible)
2. Add method to existing agent or create new agent
3. Register in `Coordinator._dispatch()` routing
4. Add validator to `CriticAgent`
5. Default toggle to OFF (opt-in only)

**Example**: Anti-leakage protocol checks for material gaps in prose without modifying existing logic.

## Safety Guarantees

- ✅ No modifications to System A
- ✅ No changes to WritingAgent output format (bullets only)
- ✅ No removal of existing functionality
- ✅ All toggles default to OFF
- ✅ Rollback via toggle disable + git revert
