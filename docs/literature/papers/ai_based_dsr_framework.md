## Core argument
As AI systems become artefacts of DSR studies, the standard Hevner (2004) / Peffers (2007) DSR frameworks require extension — specifically, AI-based artefacts introduce new evaluation challenges (non-determinism, emergent behaviour, calibration) and new design knowledge categories that must be explicitly addressed in the DSR process.

## Method
Literature review + conceptual framework development. Synthesises DSR literature (Hevner 2004, Peffers 2007, Sein 2011) with AI/ML artefact properties. Proposes an exploratory AI-based DSR framework with 4 adapted phases: problem identification → AI artefact design → iterative AI evaluation → abstracted design principles.

## Key finding
Standard DSR evaluation criteria (utility, quality, efficacy) are insufficient for AI artefacts. AI-specific evaluation dimensions required: calibration, robustness under distribution shift, explanation quality, and resource efficiency. Proposes operationalised metrics for each dimension.

## Key quote
> "AI artefacts challenge the standard DSR evaluation canon — their non-determinism and emergent behaviour demand new evaluation protocols that go beyond the utility-quality-efficacy trifecta."

## Relevance to thesis
- **Core methodological foundation** for Ch.3: directly justifies applying DSR to an AI multi-agent system
- Provides language for why the thesis's 3-level evaluation (ML accuracy + recommendation quality + agent behaviour) is methodologically correct for an AI artefact
- Cites the same canonical DSR sources (Hevner 2004, Peffers 2007) the thesis builds on — creates a clean citation chain
- Legitimises the thesis as DSR at a CBS examiner level

## Gap / limitation
Exploratory/conceptual framework — not yet empirically validated at scale. Focuses on single AI artefact evaluations, not multi-agent systems. Does not address RAM/compute constraints as a DSR evaluation dimension (gap the thesis can fill).
